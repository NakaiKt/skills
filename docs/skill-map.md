# スキルマップ（分類の正典）

このリポジトリのスキルを「どんなときに使うか（場面）」で分類・登録する唯一の正典。
`skill-curator` はこのファイルを読んで新しい知見の配置先を判定し、反映後にこのファイルを更新する。
人間もここを見れば「何のスキルがあり、各々いつ発動し、どの参照資料を持つか」が一望できる。

> 原則：スキルは「何を参照しているか（本・PDF）」ではなく「**どんなときに使うか**」で括る。
> 1冊の知見が複数スキルに分散するのは**正常**。本は素材、スキルは場面。

---

## 面（プレーン）

| 面 | 目的 | スキル |
|---|---|---|
| **業務面** | あなたの仕事そのものを助ける | `goal-design` / `improve-loop` /（将来）`team-ops` |
| **メタ面** | 業務面のスキル群を育て続ける | `skill-curator` |

---

## 分類の軸

新しい知見・場面を分類するときは、この軸に当てはめる。

### 主軸：ゴールに対して「今、何をしているか」

1. **定める** … ゴール／ビジョン／目標／指標をこれから決める → `goal-design`
2. **回す** … 決めた後、打ち手を実験し・検査し・ふりかえって適応する → `improve-loop`
3. **育てる** … 人とチームを立ち上げ・運営する → `team-ops`（将来）

### 副軸：作る / 斬る（＝スキルは分けない。各スキル内の Step 0 モード）

- **作る**：新規に立てる（発散→収束）
- **斬る**：既存の成果物を基準に照合して添削する（例：OKR添削）

副軸でスキルを割ると発動条件が全スキルと衝突するので、**必ずスキル内のモード分岐で吸収**する。

### 対象：個人 / チーム・組織（＝スキルは分けない。references を分ける）

コアロジックは対象非依存なので、対象で**スキルは割らない**。
Step 0 で対象を分岐し、`references/team.md` / `references/personal.md` を読み分ける。

---

## 新スキル新設のゲート規則

既存スキルの `references/` 追記で済むなら**新設しない**（原則こちら）。
新スキルを作ってよいのは、次の**両方**を満たすときだけ：

1. 発動条件（トリガー語）が既存スキルと**衝突しない**
2. コアロジックが既存スキルと**別物**

---

## SKILL.md を薄く保つ原則

- `SKILL.md` 本体は「目的＋Step 0 の分岐＋各モードの短い手順」の**ルーター**に留める。
- 重い中身（概念解説・チェックリスト・指標カタログ・ペルソナ）は `references/*.md` に逃がす。
- 知見の追記は原則 `references/` へ。新しいトリガー語が生まれたときだけ `description` に追記する。

## レジストリ

### `goal-design`（定める）

| 項目 | 内容 |
|---|---|
| 場面 | 企画のゴール/ビジョン、個人の四半期・月・週目標、SMART目標、KPI/指標設計、「なぜやるのか」の遡上、既存OKR/KPI/目標の添削、投資・意思決定の整理 |
| 発動条件 | 「目標/ゴール/ビジョンを決めたい」「四半期(月/週)の目標」「KPI/指標/メトリクス設計」「SMARTにしたい」「これやる意味ある?/なぜやる」「このOKR/目標を添削して」「どこに投資すべき」 |
| モード | 作る／斬る |
| 対象 | 個人／チーム・組織 |
| references | `team.md`（3階層ゴール・5指標分類・4KVA・KVMカタログ・投資レンズ・添削チェックリスト）、`personal.md`（個人への翻訳・SMART・個人の添削チェックリスト） |
| 収録知見 | EBM Guide 2024 |
| 分割トリガー | 原則なし（目標ロジックは対象非依存。永続的に1スキルでよい） |

### `improve-loop`（回す・ふりかえる）

| 項目 | 内容 |
|---|---|
| 場面 | 施策/アプローチ立案・実験、進捗評価、四半期/月/週のふりかえり、レトロ、リフレクション、「次どうする」 |
| 発動条件 | 「この施策どう進める/実験したい」「うまくいったか評価して」「ふりかえり/レトロ/リフレクション」「次どうする」「スプリント/スクラムイベント」 |
| モード | 作る（打ち手を立てる）／斬る（結果を検査する） |
| 対象 | 個人／チーム・組織 |
| references | `team.md`（実験ループ・機能=仮説・チームのふりかえりの器）、`personal.md`（型カタログ〔YWT/KPT/Gibbs/Kolb〕・認知の4点セット〔意見/経験/感情/価値観〕・反芻/省察ガード・問いの銀行・リズム設計・バイアス補正・持論化） |
| 収録知見 | EBM Guide 2024、熊平美香『リフレクション』、Kolb 経験学習、Gibbs、Driscoll、Tasha Eurich、YWT/KPT |
| 分割トリガー | 個人向け（`personal.md`）は内省の知見が充実してきた。チーム向けのアジャイル/スクラム知見が同程度に厚くなり、かつトリガー語が「自分のふりかえり/内省」vs「チームのレトロ/スプリント」に明確に割れたとき、`self-reflect` を独立させる |

| 項目 | 内容 |
|---|---|

### `skill-curator`（メタ：育てる仕組み）

| 項目 | 内容 |
|---|---|
| 場面 | 読書メモ・PDF・Notion等の知見を業務スキルに振り分けて取り込み・更新する |
| 発動条件 | 「この読書メモ/PDF/Notionをスキルに取り込んで」「知見を振り分けて」「スキルを更新して」「これどのスキルに入れる?」 |
| references | `extraction.md`（知見の道具化パターン） |
| 備考 | このファイル（skill-map.md）を読み書きして運用する |
