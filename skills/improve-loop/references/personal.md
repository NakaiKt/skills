# improve-loop / 個人向け参照

出典: Evidence-Based Management Guide 2024（Scrum.org）を**個人のふりかえり**に翻訳して道具化したもの。
※リフレクション/内省の具体知見は将来ここに追記する（現状はEBMの検査・適応ロジックが中心）。

---

## 1. 個人の実験ループ

個人の目標も「仮説→実験→検査→適応」で回す。

1. **仮説** … 「この習慣/やり方 X を試すと、自分の Y（能力・状態）が変わる」と考える。
   何をもって「効いた」とするかを先に決める。
2. **実験** … 一定期間試す。全部を完璧にやろうとせず、効くかを確かめる最小限で。
3. **検査** … Activity（やった量）ではなく Outcome（自分の変化）で見る。
   「毎日やった」ではなく「以前できなかった◯◯ができるようになったか」。
4. **適応** … 効いたなら次のゴールへ（→ goal-design）。効かなかったら、それは失敗ではなく
   次のより良い実験の材料。やり方かゴールのどちらを変えるか決める。

---

## 2. 週次 / 月次 / 四半期のふりかえり

定例のリズムで回すときの暫定手順（将来リフレクション本の知見で厚くする）。

1. **棚卸し** — この期間の打ち手・行動を「何を狙っていたか（仮説）」で並べる。
2. **検査** — 狙ったアウトカム（行動変容・能力獲得）が起きたか。起きていないならなぜかを掘る。
3. **学びの抽出** — うまくいった/いかなかった理由から、再現/回避できる要因を1つ言語化する。
4. **適応** — 次の週/月/四半期の即時戦術ゴールへ繋ぐ。ゴール自体を見直すなら goal-design へ。

**繋ぎ方**：四半期・月・週の定例では「本スキルで前期を検査 → goal-design で次期を定める」の順。
EBMの実験ループ最終段「適応（学びからゴールを見直す）」が、ちょうど2スキルの継ぎ目になる。

---

## 3. 良い失敗として扱う

期待した結果が出なかったのは悪いことではなく、学び。自分を責める材料にせず、
「次はどうすればより良くできるか」に変換する。これは EBM が明言する姿勢。
