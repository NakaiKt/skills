# improve-loop / 個人向け参照

個人のふりかえり（週次/月次/四半期の定例、1つの経験の内省、レトロ／リフレクション）を
深く回すための道具箱。**ワークの枠を埋めるだけで終わらせず、一段深掘りする**ための
前提知識をここに集める。

土台にしているのは次の3系統。混ぜて使う。

- **経験学習（Kolb）** … 経験 → 省察 → 概念化（持論化）→ 実践 のサイクル。ふりかえりの背骨。
- **実験ループ（EBM）** … 仮説 → 実験 → 検査 → 適応。打ち手を「価値の仮説」として検査する。
- **リフレクション／メタ認知** … 自分を一段上から俯瞰し、意見の裏にある経験・感情・価値観まで
  掘る。ワークを「深掘り」に変える道具（`認知の4点セット` が中心）。

> 3系統の関係：Kolb が**器（回す流れ）**、EBM が**検査の基準（成果物でなくアウトカムで見る）**、
> リフレクションが**掘る道具（浅い記述で止めない）**。型カタログ（§1）で器を選び、
> 4点セット（§2）と反芻ガード（§3）で深さを担保する、と捉えるとよい。

---

## 1. Step 0: 場面で「型」を選ぶ（型カタログ）

ふりかえりに万能の型はない。**今日の目的と持ち時間**で選ぶ。

| 型 | 向く場面 | 構造 | 深さ・時間 |
|---|---|---|---|
| **YWT** | 軽い定例・行動の棚卸し | やったこと / わかったこと / つぎやること | 浅〜中・短 |
| **KPT** | 継続改善の週次 | Keep（続ける）/ Problem（課題）/ Try（試す） | 中・中 |
| **Driscoll** | 忙しい日の最短内省 | What?（何が）/ So What?（だから何）/ Now What?（で、次は） | 浅・最短 |
| **Fun/Done/Learn** | 気力が落ちた時・楽しさの回復 | 楽しかった / 完了した / 学んだ の3円 | 浅・短 |
| **Gibbs リフレクティブサイクル** | 感情が動いた1経験の深掘り | 記述→感情→評価→分析→結論→行動計画（6段） | 深・長 |
| **Kolb 経験学習サイクル** | 経験を「自分の持論」に昇華したい | 経験→省察→概念化→実践（§7） | 深・長 |

選び方の目安：

- **軽い定例で回すリズムを保ちたい** → YWT か KPT。まず習慣化を優先。
- **強い感情が残った出来事（失敗・衝突・手応え）を消化したい** → Gibbs。感情を段として持つのが強み。
- **1回の経験を今後に効く「型」に変えたい** → Kolb（→§7 持論化）。
- **どの型でも「浅くて終わった」と感じたら** → その場で §2 の認知の4点セットを当てて一段掘る。

> 型は入口にすぎない。「型を埋めた＝ふりかえった」ではない。埋めた後に §2/§3 で深さを足す。

## 2. 深掘りのコア道具 — 認知の4点セット（意見・経験・感情・価値観）

出典：熊平美香『リフレクション（REFLECTION）』の中心ツールを道具化。
**どの型を使っていても、一段深めたいときに当てる汎用レンズ**。個人にもチームにも効く。

人の「意見（結論・主張）」の裏には、必ずそれを支える経験・感情・価値観がある。
表面の意見だけ見ていると深まらない。4点に分解すると、自分でも気づいていなかった
前提が見えてくる。

| 点 | 問い | 役割 |
|---|---|---|
| **意見** | 私は何を思った／主張したか？ | 表に出ている結論 |
| **経験** | その意見の背景にある経験・見聞きした事実は？ | 意見の根拠になった具体 |
| **感情** | そのときどんな感情が動いたか？ | 判断を裏で駆動しているもの |
| **価値観** | その意見・感情の奥にある、私が大事にしている価値観は？ | 繰り返し現れる自分の芯 |

### 使い方A：自分の意見を分解する（メタ認知）

ふりかえりで「◯◯すべきだった」「あれは良かった」と結論が出たら、そこで止めず4点に割る。

1. **意見**：出てきた結論をそのまま書く。
2. **経験**：なぜそう思う？ どの経験・事実が根拠か。抽象語（「なんとなく」「普通」）を具体に落とす。
3. **感情**：そのとき自分は何を感じていたか。感情は判断を歪める／駆動するので必ず言語化する（→§3）。
4. **価値観**：この意見・感情の奥で、自分は何を大事にしているのか。
   → ここで見えた価値観は、複数のふりかえりを跨いで**繰り返し出てくる自分の芯**になりやすい。

### 使い方B：前提を疑う（アンラーン）

見えた価値観・前提は「正しい」とは限らない。過去に有効だった前提が、今は足を引っ張ることもある。

- その価値観は**今の状況でも有効か**？ 別の見方はないか？
- 「こうあるべき」と思っている前提を1つ選び、**逆張り**してみる（もし逆が正しいとしたら？）。
- 手放してよい前提が見つかったら、それが最大の学び（＝アンラーン）。

### 使い方C：チームのふりかえりでも使う

相手の発言を4点で聴く／問い返すと対話が深まる（詳細は team.md 側にも配置）。
「その意見の背景にある経験は？」「そこにどんな感情が？」と問うと、意見のぶつかり合いが
価値観の相互理解に変わる。個人の内省と同じ道具が、そのままチームの器で効く。

## 3. 反芻（rumination）と省察（reflection）を切り分けるガード

内省は万能ではない。**やり方を間違えると、洞察ではなく反芻（ぐるぐる悩み）に落ちて逆効果**になる。
Tasha Eurich の自己認識研究が示す、内省を洞察に変えるための操作。

### 「なぜ？」を「何が？（What）」に置き換える

- **「なぜ失敗したのか」「なぜこう感じるのか」**と問うと、人は自分に都合のよい物語を作り、
  過去に沈み込みやすい（＝反芻）。
- **「何が起きたのか」「何が効いたのか」「何を変えれば次は違うのか」**と問うと、
  事実と前進に向かう（＝洞察）。
- 原則：**内省では "Why" より "What" を主語にする**。§2 の「経験」を掘るときも、
  「なぜそう思う」ではなく「どの事実からそう思う」と問うと反芻を避けられる。

### 感情の扱い

- **affect labeling（感情に名前をつける）**：「モヤモヤ」で止めず「悔しさ」「安堵」「焦り」と
  具体名を当てる。名前をつけるだけで感情の圧が下がり、判断への干渉が減る（脳科学的にも裏付けあり）。
- **事実と解釈を分ける**：「彼は私を軽視した（解釈）」と「返信が3日なかった（事実）」を分けて書く。
  ふりかえりが荒れるのはたいてい解釈を事実と思い込むとき。

### 内省しすぎの境界

- 同じ問いを3周しても新しい事実・行動が出てこないなら、それは反芻。**問いを "What/Now What" に
  切り替える**か、いったん閉じる。
- ふりかえりのゴールは「気が済むまで考える」ではなく「**次の1手が決まる**」こと。行動が1つ出たら閉じてよい。

## 4. 問いの銀行（プロンプト集）

ふりかえりの質は「どんな問いを自分に投げられるか」でほぼ決まる。粒度・目的別のストック。

### 事実の棚卸し（判断の前に必ず）

- この期間、実際に何をした／何が起きたか（記録・ログを先に見る。記憶だけで語らない → §6）。
- 予定と実際のズレはどこにあったか。

### うまくいったことを掘る

- 何がうまくいったか。**それは偶然か、再現できる要因があるか**。
- その成功を支えた自分の行動・判断は？（成功の再現条件を1つ言語化する）

### うまくいかなかったことを掘る（責めずに）

- 期待した結果は何で、実際は何だったか（アウトカムで見る → EBM）。
- 何が効かなかったか。**次に何を変えればより良くできるか**（→ "Now What"）。

### 深掘り（§2 と連動）

- その判断の背景にある経験・感情・価値観は？（認知の4点セット）
- 当たり前だと思っていた前提で、疑えるものは？（アンラーン）

### 次へ繋ぐ

- 次の週／月／四半期に**続けること／やめること／始めること**は？
- ゴール自体を見直す必要はあるか？（あれば → goal-design）

---

## 5. リズム設計（粒度で深さと問いを変える）

粒度が上がるほど、行動の記録から**パターン・前提・ゴール**へと視点を上げる。

| 粒度 | 主眼 | 型の目安 | 主な問い |
|---|---|---|---|
| **日次** | 事実の記録と感情の放流 | 一行日記 / Driscoll | 今日何が起きた？ 何を感じた？（判断はしない、記録だけ） |
| **週次** | 行動の検査と微調整 | YWT / KPT | 狙い通り動けたか？ 続ける／変えることは？ |
| **月次** | パターンの発見 | KPT ＋ 4点セット | 週次を跨いで繰り返す課題・強みは？ その奥の価値観は？ |
| **四半期** | ゴールの検査と適応 | Kolb ＋ EBM検査 | 立てたゴールに近づいたか？ ゴール自体は今も適切か？（→ goal-design） |
| **年次** | 前提の棚卸し（アンラーン） | 4点セット中心 | 手放すべき前提は？ 来年の自分の芯（価値観）は？ |

- 日次は**貯める**（判断しない）、週次以降で**読む**。日次で無理に学びを出そうとしない。
- 上位の粒度は下位の記録を素材にする。だから日次の記録が効いてくる（→§6）。

## 6. 自己評価のバイアス補正

自分のふりかえりは、放っておくと自分に都合よく歪む。先に補正装置を置く。

- **記録を先に、判断を後に**：記憶は再構成される。ログ・カレンダー・成果物を先に並べてから
  評価する。「たぶん頑張った」ではなく残った事実から始める。
- **後知恵バイアス**：「やっぱりこうなると思っていた」——結果を知った今の視点で過去を裁かない。
  当時に何を知っていたかで判断する。
- **自己奉仕バイアス**：成功は自分の実力、失敗は環境のせい、に傾く。逆の仮説も一度立てる
  （成功に運の要素は？ 失敗に自分の関与は？）。
- **直近性バイアス**：期間の最後の出来事が過大評価される。期間全体を均等に棚卸しする（→§4 事実の棚卸し）。
- **良い失敗として扱う**：期待した結果が出なかったのは悪ではなく、次のより良い実験の材料。
  自分を責める材料にせず「次はどうすればより良くできるか」に変換する（EBM が明言する姿勢）。

---

## 7. 経験の持論化（Kolbの概念化段）

単発の学びを「その時だけの反省」で終わらせず、**次に再利用できる自分の型（持論）**に昇華する。
ふりかえりが積み上がる人とそうでない人の差はここに出る。

1. **経験**：起きた具体を1つ選ぶ。
2. **省察**：何が効いた／効かなかったかを掘る（§2/§3 を使う）。
3. **概念化（持論化）**：「◯◯なときは△△するとよい」という**一般化した一文**にする。
   固有名詞を外し、他の場面でも使える形に。
4. **実践**：次に似た場面が来たら、その持論を意図的に試す（→ また経験になり、ループが回る）。

- 持論は**仮説**でよい。次の実践で反証されたら更新する（EBM の適応）。
- 溜まった持論は、繰り返し現れる**価値観**（§2）と結びつく。自分の「効き方の癖」の地図になる。

---

## 8. 繋ぎ方（goal-design への渡し）

- 四半期・月・週の定例は「**本スキルで前期を検査 → goal-design で次期を定める**」の順。
- EBM の実験ループ最終段「適応（学びからゴールを見直す）」が、ちょうど2スキルの継ぎ目。
- ふりかえりで「ゴール自体を変えるべき」と出たら、その学びを持って goal-design へ渡す。
