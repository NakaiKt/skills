# improve-loop / リフレクション（深掘りの前提知識・個人＆チーム共通）

ふりかえりのワーク（YWT/KPT/レトロ等）を**「枠を埋める」で終わらせず、一段深く掘る**ための
前提知識。`personal.md`・`team.md` の両方がここを参照する（この道具は対象非依存）。

土台：**リフレクション = メタ認知**（自分の思考・判断・感情を一段上から俯瞰すること）。
枠を埋める作業そのものより、「自分が何をどう見て・感じて・決めているか」を見る目を持つことが本体。

出典：熊平美香『リフレクション』／ Donald Schön（反省的実践）／ Chris Argyris（推論のはしご・
ダブルループ学習）／ Peter Senge（クリエイティブ・テンション）。

---

## 1. いつ省察するか — Schön の2つのタイミング

ふりかえりは「後で振り返る」だけではない。Schön は2つを区別する。

- **reflection-on-action（行為の後の省察）** … 定例ふりかえり。起きたことを後から掘る。本ファイルの型はここが中心。
- **reflection-in-action（行為の中の省察）** … やりながら「今うまくいっていない、変えよう」と
  **その場で立ち止まって修正する**こと（"thinking on your feet"）。熟達者ほどこれができる。

育て方の順序：まず on-action（定例）を習慣化 → 掘って得た持論（personal.md §7）が体に入ると、
in-action が効き始める。**ふりかえりの最終目的は「その場で修正できる自分／チーム」を作ること**。
定例は手段であって目的ではない。

---

## 2. リフレクションの5つの入口（熊平）

ふりかえり＝「経験から学ぶ」だけではない。**目的で入口が変わる**。今日はどの入口かを意識する。

| 入口 | 主な問い | 使う場面 |
|---|---|---|
| **自分を知る** | 自分の動機の源泉は？ 大事にしている価値観は？ | 迷ったとき・年次・キャリア |
| **ビジョンを形成する** | ありたい姿は？ 現状とのギャップは？ | 期の頭（→ goal-design と接続） |
| **経験から学ぶ** | この経験から何を得たか（§3 4点セット＋ Kolb） | 定例ふりかえりの主戦場 |
| **多様な世界から学ぶ** | 他者・異なる視点から何を得るか（§6 評価判断の保留） | 対話・レトロ・1on1 |
| **アンラーンする** | 手放すべき前提・古い成功体験は？（§5 ダブルループ） | 停滞したとき・環境が変わったとき |

- 「経験から学ぶ」に偏りがちだが、停滞時は**アンラーン**、対人の悩みは**多様な世界から学ぶ**が効く。
- **ビジョン形成**は goal-design の領分。ここでは「入口がある」ことだけ示し、深入りは goal-design へ渡す。

## 3. 認知の4点セット（意見・経験・感情・価値観）

熊平リフレクションの中心ツール。**どの型を使っていても、一段深めたいときに当てる汎用レンズ**。

人の「意見（結論・主張）」の裏には、必ずそれを支える経験・感情・価値観がある。
表面の意見だけ見ても深まらない。4点に分解すると、自分でも気づいていなかった前提が見えてくる。

| 点 | 問い | 役割 |
|---|---|---|
| **意見** | 何を思った／主張したか？ | 表に出ている結論 |
| **経験** | その意見の背景にある経験・見聞きした事実は？ | 意見の根拠になった具体 |
| **感情** | そのときどんな感情が動いたか？ | 判断を裏で駆動しているもの |
| **価値観** | その意見・感情の奥で大事にしている価値観は？ | 繰り返し現れる自分の芯 |

### 使い方A：自分の意見を分解する（メタ認知）

結論（「◯◯すべきだった」）が出たら止めず4点に割る。経験＝どの具体が根拠か、感情＝何を感じたか
（必ず言語化 → personal.md §3）、価値観＝奥で何を大事にしているか。価値観は複数のふりかえりを
跨いで**繰り返し出てくる自分の芯**になりやすい。

### 使い方B：チームの対話で使う

相手の発言を4点で聴く／問い返すと、意見のぶつかり合いが**価値観の相互理解**に変わる。
「その意見の背景の経験は？ そこにどんな感情が？」。対立したら**両者の4点セットを机に並べてから**
評価する（→ §6 評価判断の保留）。

> 4点セットの「経験」を掘る作業は、次の §4 推論のはしごを降りることと実質同じ。セットで使う。

## 4. 推論のはしご（Ladder of Inference / Argyris）— 飛躍を検査する

人は **事実 → 選ぶ事実 → 意味づけ → 前提 → 結論 → 行動** を一瞬で登ってしまう。
同じ出来事でも人によって結論が違うのは、どこかの段で**飛躍**しているから。

- **降りて検査する**：出した結論から逆に降りる。「どの事実から？」「どう意味づけた？」「どんな前提を置いた？」。
- **対話**：相手と食い違ったら「あなたがそう考えた事実は？」と問い、**どの段で分かれたか**を特定する。
  多くの対立は結論の是非ではなく、選んだ事実・意味づけの違いに根がある。
- **効用**：「彼は私を軽視した（＝はしごの上）」を「返信が3日なかった（＝はしごの下・事実）」まで
  降ろすと、ふりかえりが感情論から検査可能な話に変わる（→ personal.md §3 事実と解釈の分離）。

---

## 5. シングルループ / ダブルループ学習（Argyris & Schön）

学び＝誤りの検知と修正。修正には2つの深さがある。

- **シングルループ**：今の枠組みの中で**行動・打ち手を直す**（やり方を変える）。
- **ダブルループ**：**枠組み・前提・目標そのものを問い直す**（何を目指すか、何を正しいと信じているかを変える）。

ふりかえりが「次はもっと頑張る」で終わるのは**シングルループ止まり**。
「そもそもこの目標／この前提は今も正しいか」まで行くとダブルループ＝**アンラーン**。

- **EBM との接続**：実験ループの「適応」で*打ち手だけ*変えるのはシングルループ、*ゴール自体*を
  見直すのがダブルループ。両方要る。ゴール見直しに至ったら goal-design へ渡す。
- 停滞・違和感が続くときは、たいていシングルループを回し続けている。前提を1つ選んで疑う（→§6）。

---

## 6. 評価判断の保留（Suspend judgment）

新しい／他者の意見に触れたとき、反射的に正誤を裁かず**一度受け止める**。多様な世界から学ぶ前提。

- 「それは違う」と即断すると、そこで学びが止まる。まず相手のはしご（§4）と4点セット（§3）を聴く。
- 自分の**価値観**（§3）が判断を駆動していることを自覚し、いったん脇に置く。
- 反芻を避ける操作（personal.md §3 の「なぜ→何が」）と組み合わせると、自分の内省でも効く
  （自分の結論を即座に正当化せず、保留して事実に降りる）。
