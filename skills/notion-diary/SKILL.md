# Notion日記記録スキル

Katsuhiroの日記DBに1日のエントリーを記録するためのスキル。

## 日記DB情報

- **DB名**: 📕 日記
- **data_source_id**: `32dc77a9-8998-807e-876a-000b4eb5de62`
- **DB URL**: `https://www.notion.so/32dc77a98998804b9518f685200747d8`

---

## プロパティスキーマ

| プロパティ | 型 | 説明 |
|---|---|---|
| `タイトル` | title (必須) | 入力のタイトルをそのまま使用 |
| `日付` | date | 入力の日付をそのまま使用（ISO形式: `YYYY-MM-DD`） |
| `天気` | select | 入力の天気をそのままマッピング |
| `コンディション` | number | 日記の全文を読んで0〜100（10刻み）で設定（下記判断基準参照） |
| `プロンプトの種` | checkbox | 内容に知見が含まれているかどうかで判断（下記参照） |

### 天気のマッピング

入力の天気テキストを以下の選択肢に対応させる：

| 入力テキスト | Notionの選択肢 |
|---|---|
| 晴れ / 快晴 / 晴 | `☀️晴れ` |
| 曇り / 曇 / くもり | `☁️曇り` |
| 雨 / 雨天 | `☔雨` |
| 雪 / 積雪 | `☃雪` |

### コンディションの判断基準

日記の全文（概要・良かったこと・イマイチだったこと・思ったこと）を読んで総合的に判断し、**0〜100の整数（10刻み）**で設定する。0が最悪、100が最高。

| スコア | 状態の目安 |
|---|---|
| 0〜10 | 体調不良・強いネガティブ感情・落ち込みが顕著 |
| 20〜30 | 疲労・ストレス・億劫さが目立つ。気力が低め |
| 40〜50 | 可もなく不可もない。良いことも悪いこともほどほど |
| 60〜70 | ポジティブな内容が多く、前向きな感情や達成感が見られる |
| 80〜90 | 充実感・高いモチベーションが全体を通じて感じられる |
| 100 | すべてが順調で、最高に充実した1日 |

細かい判断は10点単位で行う（例: 疲れ気味だが良いこともあったなら40、ほぼ好調だが少し引っかかりがあるなら70）。

### プロンプトの種の判断基準

以下のような知見・気づきが含まれている場合は `__YES__`（チェックあり）、ない場合は `__NO__`：

- AIへの入力プロンプトとして使えそうな気づきや洞察
- 将来の何らかのまとめノートに活用できそうな思考・発見
- 技術的な学び、設計の気づき、問題解決のヒント
- 思考の枠組みや価値観の言語化
- 繰り返し参照したくなりそうな考察

単なる日常の出来事の羅列のみの場合は `__NO__`。

```markdown

- 箇条書き内容をそのまま

## 良かったこと
- 箇条書き内容をそのまま

## イマイチだったこと
- 箇条書き内容をそのまま

## 思ったこと
- 箇条書き内容をそのまま

## フィードバック
- 箇条書き内容をそのまま

## 明日やること
- 箇条書き内容をそのまま
```

**重要**: 内容は編集せず、入力のテキストをそのまま使用すること。

## ワークフロー

### 1. 入力を解析する

日記のフォーマットから以下を抽出：
- `タイトル`: そのまま使用
- `日付`: ISO形式に変換（例: `2026年3月24日` → `2026-03-24`）
- `天気`: マッピング表を参照して変換
- `コンディション`: 本文全体を読んで判断
- `プロンプトの種`: 知見の有無を判断

### 2. レコードを作成する

`notion-create-pages` で作成。`icon` と `cover` を必ず含める：

```
parent: { data_source_id: "32dc77a9-8998-807e-876a-000b4eb5de62" }
pages:
  - properties:
      タイトル: "<タイトル>"
      date:日付:start: "<YYYY-MM-DD>"
      date:日付:is_datetime: 0
      天気: "<天気の選択肢>"
      コンディション: <0〜100の整数（10刻み）>
      プロンプトの種: "__YES__" または "__NO__"
    icon: "📝"
    cover: "https://images.unsplash.com/photo-1506784365847-bbad939e9335?w=1600&q=80"
    content: |
      ## 概要
      ...（以降を本文として設定）
```

### 3. 完了を通知する

作成したNotionページのURLを添えて、以下を伝える：
- 登録したタイトルと日付
- 判断したコンディションとその根拠を一言
- プロンプトの種の判断とその根拠を一言

---

## 注意事項

- `date:日付:is_datetime` は常に `0`（日付のみ、時刻なし）
- `天気` や `コンディション` は既存の選択肢のみ使用（新規追加しない）
- ページ本文の冒頭にタイトルは不要（`properties` の `タイトル` が自動表示される）
- 「概要」より前のメタデータ部分（日付・天気・タイトル）は本文に含めない
